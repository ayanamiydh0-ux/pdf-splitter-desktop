# PDF 智能拆分工具设计方案

## 一、需求总结

### 1.1 核心需求

开发一个基于大模型的PDF智能拆分桌面应用，支持用户通过自然语言描述拆分规则，自动识别PDF文档结构并按章节拆分。

### 1.2 功能需求

#### 1.2.1 Chat交互系统
- 提供Chat对话窗口，支持用户描述拆分规则
- 支持多种输入类型：
  - 纯文字描述
  - Markdown文档（拖拽上传）
  - 图片描述（OCR识别）
- LLM解析用户需求，生成结构化拆分规则
- 对话历史管理，支持多轮交互澄清

#### 1.2.2 智能拆分策略

**策略1：书签优先（低成本）**
- 优先尝试读取PDF书签/大纲
- 直接从书签获取章节信息和页码
- 支持层级结构拆分
- 成本：0 token

**策略2：LLM智能分析（保底方案）**
- 书签为空或不足时启用
- 采用分块处理策略（支持大文档）
- LLM理解文档内容并识别章节边界
- 自动处理跨块章节，避免错误拆分
- 成本：按实际使用计算

#### 1.2.3 分块处理方案
- 默认配置：
  - 块大小：100页
  - 重叠区大小：20页
- 支持在GUI设置中调整参数
- 块间重叠确保不切断章节
- LLM保持上下文连贯性

#### 1.2.4 文件命名规则
- 优先从书签标签名获取
- 用户可在Chat中描述命名需求
- 默认规则：章节编号 + 空格 + 章节名（如"Chapter 1 Introduction"）
- 自动处理非法字符和重复文件名
- 支持自定义命名模板

#### 1.2.5 进度显示
- 实时显示分析进度：已分析页数/总页数（如 45/150）
- 显示识别到的待拆分文件数量
- 显示运行时间
- 显示消耗的token数

#### 1.2.6 LLM集成
- 支持云端API模式
- 兼容OpenAI格式的服务商：
  - OpenAI (GPT-4)
  - Kimi (Moonshot AI)
  - GLM (智谱AI)
  - DeepSeek
  - 其他兼容接口
- 用户在界面配置API Key和模型选择

### 1.3 非功能需求

#### 1.3.1 性能
- 支持大文档处理（测试目标：1.3GB, 900页）
- 分块处理避免上下文窗口限制
- 支持断点续传

#### 1.3.2 可靠性
- 完善的错误处理和恢复机制
- 自动重试失败的API调用
- 友好的错误提示

#### 1.3.3 易用性
- 直观的Chat交互界面
- 预览拆分结果，支持确认和调整
- 提供常用模板快速开始

#### 1.3.4 可扩展性
- 模块化设计，便于添加新的拆分策略
- 支持自定义LLM配置
- 策略模式，便于扩展规则

### 2.4 关键技术实现

#### 2.4.1 分块处理核心逻辑

**块划分算法：**

```python
def calculate_chunks(total_pages: int, chunk_size: int, overlap_size: int) -> List[Dict]:
    """计算分块策略"""
    chunks = []
    current_page = 1
    chunk_id = 0

    while current_page <= total_pages:
        # 读取范围（包含重叠）
        read_start = current_page
        read_end = min(current_page + chunk_size - 1, total_pages)

        # 实际处理范围（跳过重叠）
        process_start = current_page if chunk_id == 0 else current_page + overlap_size
        process_end = read_end

        chunks.append({
            'id': chunk_id,
            'read_range': (read_start, read_end),
            'process_range': (process_start, process_end)
        })

        # 下一块的起始（重叠）
        current_page = read_end - overlap_size + 1
        chunk_id += 1

    return chunks
```

#### 2.4.3 书签提取

```python
def extract_bookmarks(pdf_path: str) -> List[Dict]:
    """提取PDF书签结构"""

    reader = PdfReader(pdf_path)
    outline = reader.outline

    if not outline:
        return []

    def process_outline(outline_items, level=0):
        """递归处理书签结构"""
        chapters = []

        for item in outline_items:
            # 获取标题
            title = item.title if isinstance(item, dict) else item

            # 获取页码
            page = reader.get_destination_page_number(
                item.dest if isinstance(item, dict) else None
            ) + 1  # 转为1索引

            chapter = {
                'level': level,
                'title': title,
                'page_num': page
            }

            # 处理子书签
            if isinstance(item, dict) and item.children:
                chapter['children'] = process_outline(item.children, level + 1)

            chapters.append(chapter)

        return chapters

    return process_outline(outline)
```

```python
def generate_filename(
    chapter_title: str,
    user_rule: Optional[str] = None,
    default_rule: str = "{number} {title}"
) -> str:
    """生成文件名"""

    # 清理特殊字符
    sanitized = sanitize_filename(chapter_title)

    # 如果用户有自定义规则
    if user_rule:
        # 调用LLM生成文件名
        filename = llm_generate_filename(chapter_title, user_rule)
        return filename

    # 默认规则：章节编号 + 空格 + 章节名
    # 示例：Chapter 1 → "Chapter 1 Introduction.pdf"
    filename = default_rule.format(
        number=extract_number(sanitized),
        title=sanitized
    )

    return f"{filename}.pdf"
```

### 2.5 错误处理设计

#### 2.5.1 错误分类和应对

| 错误类型 | 优先级 | 检测方式 | 应对策略 |
|---------|--------|---------|---------|
| API Key无效/余额不足 | P1 | API返回401/429 | 友好提示+设置引导 |
| PDF读取失败 | P1 | PyPDF2异常 | 详细错误提示+解决方案 |
| API超时/限流 | P1 | 超时/429错误 | 自动重试(3次)+断点续传 |
| Token超限 | P1 | API返回token错误 | 自动缩小块大小 |
| 跨块章节错误拆分 | P2 | 重复章节名/短章节 | 检测+重试+用户确认 |
| 页码重叠/错误 | P2 | 后端验证 | 自动修正+重试 |
| 用户需求理解错误 | P2 | 文件数量异常 | 预览确认+重新分析 |
| 结果验证失败 | P2 | 页码/章节数异常 | 重试+人工介入 |
| 文件名冲突 | P3 | 重复检测 | 自动重命名 |
| 文件名非法 | P3 | 非法字符检测 | 自动清理 |
| 跨块一致性错误 | P3 | 页码不连续 | 重试分析 |
| LLM无法识别 | P3 | 返回0或1个章节 | 提示使用书签 |
| 用户中途取消 | P3 | 取消事件 | 保存当前结果 |

#### 2.5.3 断点续传机制

```python
class ChunkProcessor:
    """支持断点续传的块处理器"""

    def __init__(self, checkpoint_file: str):
        self.checkpoint_file = checkpoint_file
        self.completed_chunks = self._load_checkpoint()

    def process(self, chunks: List[Dict]) -> List[Dict]:
        """处理所有块，支持断点续传"""
        results = []

        for chunk in chunks:
            chunk_id = chunk['id']

            # 跳过已完成的块
            if chunk_id in self.completed_chunks:
                results.extend(self.completed_chunks[chunk_id])
                continue

            try:
                # 处理当前块
                chunk_result = self._process_single_chunk(chunk)

                # 保存检查点
                self._save_checkpoint(chunk_id, chunk_result)

                results.extend(chunk_result)

            except Exception as e:
                # 保存当前进度
                self._save_progress()

                # 询问用户
                should_continue = self._ask_user_continue(e)
                if not should_continue:
                    break

        return results
```

### 2.7 性能优化

#### 2.7.1 Token优化策略

| 方法 | 说明 | 节省率 |
|------|------|--------|
| 精简页面内容 | 每页只取前300-500字 | 30-50% |
| 跳过非章节页 | 扉页、纯图片页 | 10-20% |
| 只发送关键部分 | 重叠区+新内容 | 20-30% |
| 使用小模型进行预筛选 | 快速定位章节 | 40-60% |

**预期效果：**
- 原始：900页 × 1000字 = 90万token
- 优化后：约30万token
- 节省：66%

#### 2.7.2 分块优化

- 默认配置：100页/块，20页重叠
- 对于超大文档（>500页）：可以增加到150页/块
- 对于小文档（<100页）：直接不分块处理
